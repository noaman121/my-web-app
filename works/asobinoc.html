学内ネットワーク運用

HOMENOC upstream / BGP / peering

BGPPeeringOSPFVLANACLHSRPL3DNSMonitoringWLC

overview

所属する専門学校では、HOMENOC から提供される回線を使って学内ネットワークを学生だけで運用しています。私はその運営メンバーの一人として、経路制御からサービス提供まで実運用の一部を担当しています。

本番環境のため、設定変更は検証 → 影響範囲の確認 → 投入という手順を徹底し、几帳面に確認を重ねながら作業しています。

role / 担当業務

  • BGPによる経路制御・上流(HOMENOC)とのピアリング設定の確認・保守
  • ルータ / L3スイッチのコンフィグ管理とバックアップ
  • 学内向けDNSサービスの運用
  • ネットワーク監視(死活・トラフィック)の構築・確認
  • 障害発生時の一次切り分けと記録

network overview

Internet │ ┌────┴────┐ │ HOMENOC │ upstream (BGP peering) └────┬────┘ │ eBGP ┌────┴────┐ │ Border │ 経路制御 / フィルタ │ Router │ └────┬────┘ │ iBGP / OSPF ┌────┴────┐ │ Core L3 │──── DNS │ Switch │──── Monitoring └────┬────┘ │ 各VLAN (教室 / サーバ / 管理)

運用で意識していること

  • 設定投入前に必ず差分とコマンドを見直し、影響範囲を確認してから反映する
  • 障害時は show 系コマンドで切り分けを行い、原因を一つずつ潰す
  • 仕組みを理解した上で、定型作業はスクリプト・自動化に置き換える

実機演習で得たスキル

授業外にもLab室のCisco実機・WLCを使って、VLAN・ACL・OSPF・HSRPなどの構築と検証を繰り返しています。学内ネットワークの運用と並行することで、設定の意図やコマンドの挙動を体感として理解できるようになりました。
