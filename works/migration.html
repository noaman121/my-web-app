migration

クラウド→オンプレ移行

AWS EC2 → 自宅 Proxmox

AWS EC2ProxmoxCloudflare TunnelDockerNginxGrafanaPrometheus

overview

本番運用していたポートフォリオサイトと監視環境は、もともとAWS EC2上にDocker・Nginxで構築していました。継続的な課金が発生する点と、自宅にProxmox環境を構築できたことから、同等の構成を自宅サーバへ移行しました。

移行前後の構成

環境AWS EC2 (Ubuntu) → 自宅 Proxmox VE 上の Ubuntu VM
公開方式EC2 + セキュリティグループでポート開放 → Cloudflare Tunnel(ポート開放なし)
アプリ実行Docker + Nginx (構成は概ね踏襲)
デプロイgit push → サーバ側 git hook で自動 build / 再起動
監視Grafana / Prometheus による死活・リソース監視
コストEC2課金を停止し、自宅電源のみに集約

構成図

利用者 │ HTTPS ▼ ┌─────────────────┐ │ Cloudflare │ ポート開放なしで公開 │ Tunnel / Edge │ └────────┬────────┘ │ outbound only ┌────────┴────────┐ │ Proxmox VE (自宅) │ │ ┌────────────┐ │ │ │ Ubuntu VM │ │ │ │ Docker/Nginx│ │ │ │ Grafana │ │ │ └────────────┘ │ └─────────────────┘

工夫した点

  • EC2でポート開放していた構成を見直し、Cloudflare Tunnelでアウトバウンド接続のみで公開する方式に変更し、攻撃面を縮小
  • git push をトリガーにした自動デプロイの仕組みは移行後も維持し、運用負荷を変えずに環境だけを切り替え
  • 移行後もGrafanaでサーバーリソースを継続監視し、移行前後でのレスポンスや負荷の差分を確認

結果

AWSの月額課金を停止しつつ、サイトの可用性・デプロイ運用フローを維持したまま自宅環境への移行を完了しました。今後は冗長化やバックアップ体制の整備を進めていく予定です。
